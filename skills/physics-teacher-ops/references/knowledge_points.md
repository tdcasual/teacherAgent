# Knowledge Point Lifecycle

Knowledge points can start empty and be incrementally curated.

## Status

- draft: proposed by system or teacher during analysis
- confirmed: approved by teacher, enters official taxonomy

## Rules

- Allow questions with no knowledge point.
- Propose draft knowledge points when patterns repeat.
- Request confirmation before promotion to confirmed.
- Track mapping changes for re-analysis.

## Suggested Fields

- kp_id
- name
- status (draft/confirmed)
- parent_kp(optional)
- notes(optional)
